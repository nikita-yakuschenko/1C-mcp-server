## Требования к HTTP-сервису / API (обязательно при реализации)

При разработке или доработке модуля HTTP-сервиса для заявок на расходование средств учитывать следующее, чтобы не забыть:

### Даты
- **Дата документа** — всегда выводить стандартный реквизит документа **Дата** (ЗС.Дата) в ответе, в т.ч. в short. В JSON — поле «ДатаДокумента».
- **Единый формат дат** — все даты в ответе в формате **dd.MM.yyyy** (без времени): ДатаДокумента, ДатаРасхода, ДатаСменыСтатуса, даты в ДокументОснование и в связанных документах.

### Поля во всех режимах
- **Комментарий** — выводить во всех режимах (short, базовый, full). Реквизит: Комментарий.

### Статус (Состояние)
- **Статус** — в JSON выводить представление реквизита **Состояние** (ПРЕДСТАВЛЕНИЕ(ЗС.Состояние)). В short — только Статус.
- При не short дополнительно: **КтоИзменилСтатус** (ПРЕДСТАВЛЕНИЕ(ЗС.ИзменилСтатус)), **ДатаСменыСтатуса** (если в конфигурации есть реквизит ВремяИзмененияДокумента — иначе пустая строка).

### Вложения и связанные документы
- **Вложения** — при **full=true** обязательно загружать из **Справочник.ХранилищеДополнительнойИнформации** (ХДИ.Объект = Ссылка на заявку): ИмяФайла и Данные (Base64). Блок full не должен зависеть от short: при full=true добавлять СвязанныеДокументы и Вложения в ответ **независимо** от параметра short (иначе при вызове с full=true и без short=false вложения не попадут в ответ).
- **Связанные документы** при full: **ДокументОснование** (Тип, Номер, Дата, НомерДата); список документов из **РегистрНакопления.ЗаявкиНаРасходованиеСредств** по измерению ДокументРасчетовСКонтрагентом (уникальные ссылки, в ответе — Тип, Номер, Дата, НомерДата).

### Режимы по умолчанию
- **short** — по умолчанию **true** (краткий ответ), если параметр не передан.
